# Swagger And OpenAPI

OpenAPI is a specification for describing REST APIs in a standard machine-readable format.

Swagger is a family of tools around OpenAPI. The most common one is Swagger UI, a browser page where developers can read and try API endpoints.

Why it matters:

- Frontend developers can understand backend endpoints.
- Backend developers can test APIs manually.
- API contracts become easier to review.
- Documentation stays closer to the code.

In later phases, we will generate or maintain OpenAPI docs for endpoints such as products, authentication, cart, and orders.

Example future URL:

```text
http://localhost:8080/swagger/index.html
```

CV talking point:

> Documented REST APIs with Swagger/OpenAPI to improve API discoverability and collaboration.
